# yossy-ac.com セキュリティ・不具合調査 計画

対象: `https://yossy-ac.com`（ブラックボックス／低侵襲）  
実施日: 2026-04-09

## フェーズ 1: 受動的確認（実施済み）

- 主要パスの HTTP ステータス・リダイレクト・代表ヘッダの取得
- 未認証時 `/auth/status` の応答形の確認
- ログインフォーム構造（POST 先、CSRF、student/staff タブ）の確認

## フェーズ 2: 認可の整合性（テストアカウント・実施済み）

- 運営提供のテストアカウントで **通常ログイン**（student モード）のみ実施
- ログイン後の `Set-Cookie` 属性の記録（HttpOnly / SameSite / Secure の有無）
- `/auth/status` と保護パス `/student/diary/` の HTTP ステータス・本文の突合

## フェーズ 3: 実施しない項目（ポリシー）

以下は悪用転用リスクのため **本環境からは実行しない**。社内ソースレビューまたは隔離ステージングで実施すること。

- セッション署名の破損・改ざんによる **不正昇格の試行**
- 他ユーザの **パスワード変更・上書き** の試行
- 認可バイパス用ペイロードの列挙

## フェーズ 4: ソース側での是正（担当者作業）

- `/student/diary/` の **期待ロール**（student / student_files / staff）を仕様と照合
- ミドルウェア・ルートハンドラで **同一のロール定数** を参照しているか確認
- `portal_auth` のペイロードとサーバ側検証（署名・サーバセッション）の監査
